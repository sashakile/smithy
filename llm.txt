# smithy

> Progressive Differentiation Framework for Adaptive Agent Architecture
> "A2A tells agents how to talk. Smithy tells agents how to think cheaper."
> **Status:** Specification phase — no implementation yet. Last updated: 2026-03-10.

## What It Is

Smithy reduces production AI agent costs by 50–97% by progressively differentiating
agent components from expensive LLM calls (P4, ~$0.50/call) to cheap deterministic
code (P1, ~$0/call, <1ms), with validated, reversible transitions at each step.

Three deliverables:
- `mr` CLI — analyze, propose, and execute differentiation
- DR (Differentiation Runtime) — Clojure orchestration engine (Integrant, Malli, core.async)
- Progressive Differentiation Model — grounded in cell biology metaphor

## Potency Levels

| Level | Implementation | Latency | Cost |
|---|---|---|---|
| P4: Pluripotent | Full LLM agent, multi-turn | 2–30s | ~$0.50 |
| P3: Multipotent | Constrained LLM, structured output | 200ms–2s | ~$0.05 |
| P2: Committed | ONNX classifier, Drools+PMML | 5–50ms | ~$0.001 |
| P1: Differentiated | Drools DRL/DMN, lookup, deterministic code | <1ms | ~$0 |

## Core Type System (v0.1.0)

Key types (all ≤7 required fields):
- `Potency` — enum [:P1 :P2 :P3 :P4]
- `Genome {:in schema :out schema}` — component contract, invariant across potency
- `Decision {:value any? :confidence double}` — DECIDE output
- `Fault {:origin :kind :message :retry? :retry-after-ms?}` — error type
- `Cell {:id keyword :genome Genome}` — component identity (immutable)
- `Fate {:cell-id :potency :threshold :version}` — runtime state
- `Expression {:cell-id :potency :engine :ref}` — DECIDE implementation
- `Wiring {:cell-id ...}` — operational config (cascade, deps, middleware, effects)

Five built-in engines: `:drools`, `:onnx`, `:llm`, `:lookup`, `:clojure`

## Pipeline

```
RECEIVE → DECIDE → ACT → EMIT
```

Only DECIDE varies by potency. RECEIVE normalizes input to schema-defined keys.
Cascade lives outside pipeline; tries P1→P4 until confidence ≥ threshold, returns
best sub-threshold Decision if all levels fail.

## Key CLI Commands

```sh
mr init                                    # initialize project
mr plan <component>                        # analyze history, propose differentiation
mr differentiate <component> --to P1       # execute differentiation
mr reprogram <component> --to P4           # reverse differentiation
mr shadow <component> --from P4 --to P1    # validate in parallel on live traffic
mr review <component>                      # approve/reject proposal
mr status                                  # DR health and component states
```

## Integrations

- Spring AI: `SmithyDifferentiationAdvisor` intercepts ChatClient calls
- MCP: exposes components as tools (opt-in via `:mcp true` in Wiring)
- A2A: opaque black box — callers never know if P1 or P4 answered
- LangGraph/Python: HTTP sidecar at `POST /v1/decide`
- Kafka: core.async channels backed by Kafka topics
